## 🎯 Project Architecture

### **Design Token Foundation**
- **Mathematical Grid**: 8px spacing system
- **Typography Scale**: 1.2 ratio (Minor Third)
- **Consistent Borders**: 4px to 20px radius scale
- **Smooth Transitions**: 150ms to 500ms timing
- **Semantic Colors**: Context-aware color system

### **Component Pattern Library**
- **Button System**: Primary/secondary/ghost variants
- **Card System**: Flexible container patterns
- **Layout System**: Responsive grid and flex patterns
- **Interactive System**: Hover effects and animations
- **Badge System**: Semantic status indicators

### **Semantic Utility Classes**
```css
/* Layout */
container-responsive, folio-layout, gallery-grid-large
nav-adaptive, stack-vertical, stack-horizontal

/* Components */
btn btn-primary/secondary/ghost, card card-body
badge badge-primary, interactive, overlay

/* Typography & Spacing */
text-xs/sm/base/lg/xl/2xl, font-medium/semibold
p-4/6/8, m-4/6/8, gap-2/3/4/6, rounded-md/lg/xl
```

## 🛠️ Development Workflow

### **For Component Development**
1. Use semantic utility classes from the design system
2. Apply design tokens for consistent spacing/typography
3. Follow established component patterns (button, card, badge)
4. Validate builds to ensure zero regressions
5. Document any new patterns or utilities

### **For Feature Development**
1. Leverage existing component library
2. Build on established layout patterns
3. Use design tokens for consistent styling
4. Test across responsive breakpoints
5. Maintain semantic class naming conventions
